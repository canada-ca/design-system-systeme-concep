Contribute to the GC Design System (Draft)

The GC design system needs to be constantly improving. The goal of this framework is to foster continuous improvement of the guidance through experimentation. It creates a space for departments to experiment and innovate, and contribute their findings for teh benefit of the whole GC web community.

Identifying a needed improvement

While designing content for Canadians or while testing your current design, you may determine that either:

  • an existing component, pattern or template needs to be improved
  • a new component, pattern or template is needed to fill a gap in the design system

Consult the backlog the backlog and the experiments in progress

The first step is to find out if the improvement has already been identified. To d so, consult the backlog and the experiments in progress:

  • Experiments in progress

Design: from prototype to stable

Initial design and prototype

As soon as some form of prototype exists, it can be moved from the backlog to the "Experiments in progress" project board.

Before participating in a design round, verify if there's already an institution leading a design round.

Alpha

To go from early prototype to Alpha, the prototype needs to be accompanied with documentation that includes at the minimum:

  • a hypothesis of how to solve the issue
  • establish a timeframe for completing the work
  • a design or a coded prototype

You can start using this documentation template, even though you don't need to fill every section - placeholder for documentation template).

The Alpha phase is used to test the hypothesis, conduct research, include accessibility considerations, and code a working example for testing.

Beta

To go from Alpha to Beta, the following must be done:

  • have a working coded example
  • provide any CSS/JS overrides needed to implement in the latest WET theme for Canada.ca

  • have a working coded example

  • provide some points on "When to use"
  • provide bilingual documentation, using the documentation template

Use the Beta phase to scale up the experiment and fix any issues you encounter.

Beta updates are available through the main design system library, and departments can use the beta versions, as long as they consider it an experiment and are willing to report back on findings.

Iterate: do other design rounds

Once a new or updated pattern has moved to stable, it becomes the new baseline guidance.

If this round hasn’t solved all the problems identified, or if new issues arise, the pattern or template can be added back to the backlog and be the subject of another design round.
