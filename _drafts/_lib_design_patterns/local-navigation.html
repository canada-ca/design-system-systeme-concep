Local navigation: Design pattern

How to enable navigaton to related pages to help task success.

Available components

1. Unordered multi-page navigation

Use unordered multi-page navigation for pages that are related but not ordered in a sequential manner.

2. Sequential steps

Use sequential steps for related pages that should be looked at in a logical sequential order. For example, for service initiation.

3. Tabbed interface

Use a tabbed interface to group different aspects of the same content.

4. Previous and next buttons

Use previous and next buttons to help navigating to the next or previous step.
