Service initiation template: Alpha design pattern

  • Last updated: Aug 13, 2019
  • Progress: Alpha - draft
  • Design type: Template
  • Lead: Canada Revenue Agency

Service initiation are used to present services on Canada.ca. The template offers a local navigation pattern to help find answers related to the service, intended to offer a consistent experience across GC services.

The navigation can either be ordered or unordered, depending on the nature of the service. Particularly for services where the top task is applying, it is suggested to use an ordered navigation that explains who can apply, how to apply, and what an applicant must do after applying.

When to use

  • Use for services for which a set of pages can support findability of various answers a visitor is likely to need about eligibility or after they apply
  • Consider using for non-transactional services such as providing advice about diseases or travel destinations
  • Can also be adopted to group together services that, although separate, are associated by visitors' mental models, such as changing an address and changing direct deposit details

What to avoid

For a service that is simple and does not need to support answering many questions, consider designing the task in a single page.

Information architecture and breadcrumb options

Option 1: Overview page

When to use: An overview page should generally be included to introduce what it is for and to provide a shortcut to the task page they need, whether it is about applying, after they apply or how to contact.

Breadcrumb: The task pages should all have the overview page as their parent page.

Titles: Do not use the word 'overview' on the overview page. It should simply be the name of the service. This will also be the label for the first H1 on each task page (see double H1s).

Option 2: No overview

When to use: For certain services, the overview page can be ommited. Consider ommitting it when:

  • the user should always read through the pages in the sequence provided
  • the pages each provide a distinct enough task that Canadians usually only want to complete 1 of them during a visit
  • the convention for the tasks is to start

Breadcrumb: The task pages should point back to the topic or navigation page above the service.

Titles: The double H1 pattern should still be used across the task pages. (See double H1s).
